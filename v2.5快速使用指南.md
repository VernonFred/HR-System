## 📊 三大新功能使用指南

### 功能1：岗位匹配度雷达分析

#### 使用场景
**问题：** 这个候选人的 EPQ 分数与岗位要求匹配吗？  
**解决：** 雷达图直观展示实际值和理想值的对比

#### 操作步骤
1. 登录 admin.html（密码：`epq_admin_123`）
2. 滚动到"数据可视化分析"区域
3. 点击第一个标签 **"岗位匹配度分析"**
4. 查看雷达图，实线=实际平均值，虚线=理想值

#### 如何解读
- **实线接近虚线** → 匹配度好 ✅
- **实线远离虚线** → 偏差较大 ⚠️
- **关注 E/N/P/L 四个维度** → 哪些维度需要关注

#### 示例
```
Java开发工程师：
- 理想 E=10（专注），实际 E=15（偏外向）→ 可能影响专注度
- 理想 N=5（稳定），实际 N=12（偏敏感）→ 抗压能力需关注
- 理想 P=10（逻辑），实际 P=9（接近）→ 逻辑能力匹配 ✅
- 理想 L=16（诚实），实际 L=18（很诚实）→ 可靠性强 ✅
```

### 功能2：同岗位竞争力排名

#### 使用场景
**问题：** 这么多候选人，哪些是优质的？怎么快速筛选？  
**解决：** 五级评分系统，快速识别 S/A 级候选人

#### 操作步骤
1. 点击第二个标签 **"竞争力排名"**
2. 查看饼图，显示 S/A/B/C/D 五个等级的分布
3. 关注 S 级（Top 10%）和 A 级（10-30%）候选人

#### 等级说明
| 等级 | 百分位 | 建议 | 颜色 |
|------|--------|------|------|
| S 级 | Top 10% | 优秀，强烈推荐面试 | 绿色 |
| A 级 | 10-30% | 良好，建议优先面试 | 蓝色 |
| B 级 | 30-60% | 中等，可以考虑 | 紫色 |
| C 级 | 60-85% | 一般，需更多评估 | 橙色 |
| D 级 | 85-100% | 需提升，谨慎考虑 | 红色 |

#### 筛选策略
```
招聘紧急：
→ 优先面试 S/A 级
→ 节省筛选时间

招聘不紧急：
→ S/A/B 级都可以考虑
→ 综合其他因素（经验、学历等）

招聘要求高：
→ 只面试 S 级
→ 确保质量
```

#### 综合匹配分说明
- **80-100分**：优秀，与岗位高度匹配
- **60-79分**：良好，基本符合岗位要求
- **40-59分**：一般，需谨慎评估
- **0-39分**：不太匹配，建议慎重

### 功能3：候选人画像与发展建议

#### 使用场景
**问题：** 这个候选人是什么性格？怎么准备面试问题？  
**解决：** 8 种性格类型分析，提供面试建议

#### 操作步骤
1. 点击第三个标签 **"候选人画像"**
2. 查看柱状图，了解团队性格多样性
3. 点击表格中的任意候选人记录
4. 查看右侧详情抽屉

#### 八大性格类型

| 类型 | 特征 | 适合岗位 | 颜色 |
|------|------|----------|------|
| 沉稳型 | 情绪稳定、可靠 | 开发、财务 | 蓝色 |
| 活力型 | 外向开朗、积极 | 销售、运营 | 橙色 |
| 理性型 | 逻辑严谨、独立 | 开发、分析 | 紫色 |
| 敏感型 | 细腻敏感、观察 | 设计、HR | 粉色 |
| 社交型 | 善于交际、协调 | 销售、PR | 绿色 |
| 创意型 | 富有创意、活跃 | 设计、策划 | 橙红色 |
| 谨慎型 | 细心谨慎、完美 | 财务、质检 | 青色 |
| 竞争型 | 竞争意识、果敢 | 销售、管理 | 红色 |

#### 详情抽屉使用

**1. 性格画像区域**
- 显示性格类型（如：沉稳型）
- 显示综合匹配分（彩色，80+绿色，60-79橙色，<60红色）

**2. EPQ 雷达图对比**
- 蓝色实线 = 候选人实际值
- 绿色虚线 = 岗位理想值
- 对比查看匹配程度

**3. 优势特质（绿色标签）**
- 显示候选人的优势
- 例如：情绪稳定、逻辑思维、抗压能力

**4. 风险点（黄色标签）**
- 显示需要关注的点
- 例如：可能不够主动、创新性偏弱

**5. 面试建议列表**
- 根据性格类型和匹配分
- 提供具体的面试建议
- 例如：考察主动性、创新能力

## 💼 实战案例

### 案例1：招聘 Java 开发工程师

**背景：** 有 15 个候选人，需要筛选 3 人面试

**步骤：**

1. **初筛（竞争力排名）**
   ```
   查看"竞争力排名"图表：
   - S 级：2 人（优先面试）
   - A 级：4 人（可以考虑）
   - B 级：5 人
   - C/D 级：4 人（排除）
   ```

2. **细看S级（岗位匹配度分析）**
   ```
   查看"岗位匹配度分析"雷达图：
   - 候选人A：E/N/P/L 都接近理想值 ✅
   - 候选人B：N 偏高（情绪化），需关注 ⚠️
   
   决策：候选人A 优先，候选人B 备选
   ```

3. **准备面试（候选人画像）**
   ```
   点击候选人A记录，查看详情：
   - 性格类型：沉稳型
   - 优势：情绪稳定、专注力好、逻辑思维
   - 风险：可能不够主动、创新性偏弱
   
   面试准备：
   - 重点考察：主动性、创新能力
   - 问题设计：讲述一次主动承担任务的经历
   - 关注：是否有技术创新或优化经验
   ```

4. **补充A级候选人**
   ```
   继续看A级中的2个：
   - 候选人C：匹配度75分，社交型
   - 候选人D：匹配度72分，理性型
   
   决策：候选人D更适合（理性型适合开发）
   ```

**最终：** 面试候选人A、候选人B（备选）、候选人D

### 案例2：招聘销售经理

**背景：** 有 10 个候选人，需要筛选 2 人面试

**步骤：**

1. **初筛（竞争力排名）**
   ```
   S 级：1 人（张三）
   A 级：3 人（李四、王五、赵六）
   ```

2. **查看画像（候选人画像）**
   ```
   张三：活力型，E=19（很外向）✅
   李四：社交型，E=16（外向）✅
   王五：沉稳型，E=9（内向）❌ 不太适合销售
   赵六：竞争型，E=18，P=17（很高）⚠️ 需关注团队合作
   ```

3. **详细对比**
   ```
   张三 vs 赵六：
   - 张三：N=5（抗压强），P=12（平衡），建议优先
   - 赵六：N=6（抗压强），P=17（竞争性很强，可能过于激进）
   
   决策：
   - 第一人选：张三（S级，活力型，各方面平衡）
   - 第二人选：李四（A级，社交型，沟通能力强）
   ```

**最终：** 面试张三、李四

## ⚡ 快捷操作

### 键盘快捷键
- **Cmd+R** - 刷新页面
- **Cmd+S** - 保存文件
- **ESC** - 关闭详情抽屉

### 快速筛选
```bash
# 1. 按岗位筛选
选择岗位下拉框 → 选择目标岗位

# 2. 按日期筛选
点击"最近7天"或"最近30天"

# 3. 关键词搜索
输入姓名或手机号 → 实时筛选

# 4. 分数区间筛选
输入 E/N/P/L 最小值和最大值 → 点击"应用筛选"
```

### 批量操作
```bash
# 1. 全选当前页
勾选表头的复选框

# 2. 批量删除
勾选记录 → 点击"删除选中(N)"

# 3. 批量导出
勾选记录 → 点击"导出选中"
```

## 🤔 常见问题

### Q1: 为什么我的页面还是旧版本？
**A:** 您需要点击编辑器中的 **"Keep All"** 按钮接受代码更新，然后保存并刷新浏览器。

### Q2: 如何查看候选人的详细画像？
**A:** 点击表格中的任意候选人记录，右侧会滑出详情抽屉，展示完整的性格画像。

### Q3: 综合匹配分是怎么计算的？
**A:** 
- 岗位匹配度（40%）- 与理想模型的契合度
- EPQ 平衡度（30%）- 四个维度是否过度极端
- 诚实度（20%）- L 分数评估
- 完成质量（10%）- 基准分

### Q4: S/A/B/C/D 等级是什么意思？
**A:** 
- S 级（Top 10%）：优秀
- A 级（10-30%）：良好
- B 级（30-60%）：中等
- C 级（60-85%）：一般
- D 级（85-100%）：需提升

### Q5: 性格类型有哪些？
**A:** 8 种：沉稳型、活力型、理性型、敏感型、社交型、创意型、谨慎型、竞争型

### Q6: 雷达图中的两条线是什么？
**A:** 
- 蓝色实线：候选人实际 EPQ 分数
- 绿色虚线：岗位理想 EPQ 分数

### Q7: 如何自定义岗位理想模型？
**A:** 当前版本需要在代码中修改 `JOB_IDEAL_PROFILES` 对象，未来版本会提供可视化配置。

### Q8: 没有数据怎么办？
**A:** 
```bash
open debug.html
# 点击"生成测试数据"
# 或者在 test.html 中完成一次测试
```
