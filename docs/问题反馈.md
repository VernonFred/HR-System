## 🔧 导出图片居中问题 - 已修复 ✅

**问题描述**：
在导出的PNG图片和PDF文件中，总匹配度分数圆圈以及EPQ的四个维度圆圈中的分数、文字和图标都没有正确居中显示。

**根本原因**：
`html2canvas` 对复杂的CSS布局（flexbox, transform, absolute positioning）渲染支持有限，导致元素位置偏移。

**解决方案**：
1. ✅ 安装了 `dom-to-image-more` 库（对CSS支持更好的DOM转图片库）
   ```bash
   npm install dom-to-image-more
   ```

2. ✅ 修改了 `CandidatePortraitCard.vue` 中的导出逻辑：
   - 导入 `dom-to-image-more`
   - 将 `exportAsPNG()` 函数改用 `domtoimage.toPng()`
   - 将 `exportAsPDF()` 函数改用 `domtoimage.toPng()` 生成图片后再转PDF
   - 简化了PDF分页逻辑，使用图片偏移方式实现多页显示

**技术优势**：
- ✅ 更好的 flexbox 布局支持
- ✅ 更准确的 transform 渲染
- ✅ 更精确的绝对定位支持
- ✅ 高质量的图片输出

**测试步骤**：
1. 刷新前端页面（确保新代码生效）
2. 打开任意候选人的画像
3. 点击导出PNG和PDF
4. 检查导出文件中的圆圈元素是否居中显示
