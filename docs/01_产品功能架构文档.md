### 2. 三大核心功能模块

#### 🎯 模块一：智能画像生成

**功能定位**: AI驱动的候选人全方位评估

**核心流程**:
```
候选人完成专业测评
        ↓
HR上传简历（可选）
        ↓
AI多维度融合分析
        ↓
生成可视化人才画像
```

**输出内容**:
1. **人格特质雷达图**
   - 基于专业测评（MBTI/DISC/EPQ）
   - 多维度可视化展示
   - 不同测评类型自适应

2. **岗位胜任力评估**
   - AI从测评+简历中推断能力
   - 8大核心能力维度
   - 每项能力附带行为预测

3. **优势与风险识别**
   - 亮点：候选人的独特优势场景
   - 风险：可能"翻车"的场景
   - 面试验证点：HR应重点追问的问题

4. **岗位匹配分析**
   - 推荐岗位：最适合的场景+上级搭配
   - 不推荐岗位：避雷指南
   - 发展建议：成长路径

5. **综合评价**
   - AI生成的深度洞察分析
   - 3段式结构化陈述
   - 提供行为模式预测

---

#### 🎨 模块二：岗位画像配置

**功能定位**: 为AI提供岗位标准，提升匹配准确度

**核心场景**:
```
HR创建岗位画像
        ↓
配置能力维度要求
        ↓
AI匹配时自动引用
        ↓
推荐结果更精准
```

**功能亮点**:
1. **三种创建方式**
   - 手动配置：自定义能力维度和权重
   - 导入JD：AI解析岗位描述自动生成
   - 导入简历：从优秀员工简历反推

2. **AI辅助配置**
   - JD解析：提取关键能力要求
   - 简历分析：识别岗位共性特征
   - 智能推荐：建议能力维度

3. **动态生效**
   - 配置修改立即生效
   - 无需重启系统
   - 历史画像不受影响

#### 📊 模块三：专业测评中心

**功能定位**: 提供标准化测评工具

**支持测评类型**:
1. **MBTI (迈尔斯-布里格斯性格分类)**
   - 4个维度：E-I, S-N, T-F, J-P
   - 16种人格类型
   - 适用场景：团队协作、领导风格

2. **DISC (行为风格测评)**
   - 4个维度：支配型D、影响型I、稳健型S、服从型C
   - 工作风格分析
   - 适用场景：销售、管理、客服

3. **EPQ (艾森克人格问卷)**
   - 4个维度：外向性E、神经质N、精神质P、掩饰性L
   - 心理健康筛查
   - 适用场景：高压岗位、心理素质要求高的岗位

**测评管理**:
- 问卷库管理：增删改查
- 分发链接：生成测评链接
- 进度追踪：查看完成情况
- 结果查看：实时展示测评数据

## 🎯 核心功能详解

### 功能1: AI驱动画像生成

#### 1.1 多级分析模式

**分析级别**:
- **深度分析 (Pro)**: 使用Qwen2.5-32B，默认模式，适合80%场景
- **专家分析 (Expert)**: 使用DeepSeek-R1，深度推理，适合关键岗位
- **快速分析 (Normal)**: 使用Qwen2.5-7B，快速兜底

**级别对比**:

| 维度 | 深度分析 | 专家分析 |
|-----|---------|---------|
| 模型 | Qwen2.5-32B (7秒) | DeepSeek-R1 (15秒) |
| 洞察深度 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 适用场景 | 通用岗位 | 管理/核心岗位 |
| 推荐理由 | 准确+快速 | 深度+预测 |

**用户选择**:
- HR可在画像卡片头部切换分析级别
- 实时重新生成，即时查看差异
- 不影响原始测评数据

#### 1.2 数据融合机制

**融合维度**:
```
┌────────────────────────────────────────┐
│           AI 画像生成引擎               │
├────────────────────────────────────────┤
│                                        │
│  输入1: 专业测评数据 (必需)            │
│  ├─ 人格特质 (MBTI/DISC/EPQ)          │
│  └─ 行为倾向                          │
│                                        │
│  输入2: 简历数据 (可选)                │
│  ├─ 工作经历                          │
│  ├─ 项目经验                          │
│  ├─ 技能列表                          │
│  └─ 教育背景                          │
│                                        │
│  输入3: 岗位需求 (可选)                │
│  ├─ 能力维度要求                      │
│  ├─ 岗位画像配置                      │
│  └─ 目标岗位名称                      │
│                                        │
│  ↓↓↓ AI 多维度融合分析 ↓↓↓           │
│                                        │
│  输出: 深度洞察人才画像                │
│  ├─ 性格特质分析                      │
│  ├─ 岗位胜任力评估                    │
│  ├─ 优势风险识别                      │
│  ├─ 岗位匹配建议                      │
│  └─ 综合评价                          │
│                                        │
└────────────────────────────────────────┘
```

**融合策略**:
1. **单测评模式**
   - 依赖：测评数据
   - 准确度：75%
   - 适用：快速初筛

2. **测评+简历模式**
   - 依赖：测评+工作经历
   - 准确度：85%
   - 适用：正式候选人

3. **全量数据模式**
   - 依赖：测评+简历+岗位画像
   - 准确度：90%+
   - 适用：关键岗位

#### 1.3 交叉验证机制

**功能定位**: 提升画像可信度

**验证逻辑**:
```
候选人完成多个测评 (如 MBTI + DISC + EPQ)
        ↓
AI 提取共同维度 (如 "外向性")
        ↓
计算一致性得分
        ↓
识别矛盾点
        ↓
输出置信度: 高/中/低
```

**可信度判定**:
- **高置信度 (≥80)**: 各测评高度一致，结果可靠
- **中置信度 (60-80)**: 部分维度有差异，需注意
- **低置信度 (<60)**: 存在明显矛盾，建议重测或面试验证

**展示位置**:
- 评分详情弹窗 → 交叉验证Tab
- 显示一致性得分、维度检查、矛盾识别

### 功能3: 简历智能解析

#### 3.1 解析能力

**提取内容**:
- 基本信息：姓名、联系方式、求职意向
- 教育背景：学校、专业、学历、时间
- 工作经历：公司、职位、时间、职责、成果
- 项目经验：项目名称、角色、技术栈、成效
- 技能列表：专业技能、语言能力、证书

**AI深度分析**:
- 核心优势：从经历中推断，非简历自述
- 潜在风险：识别空白期、跳槽频率
- 职业轨迹：晋升速度、行业选择、发展规律
- 工作风格：偏执行/策略、独立/协作
- 稳定性评估：跳槽规律、稳定性预测
- 面试焦点：需要重点验证的疑点

#### 3.2 简历质量评分

**评分维度**:
1. **完整度 (40%)**
   - 检查：基本信息、教育、工作、技能、项目
   - 评分：0-100

2. **逻辑性 (30%)**
   - 检查：时间连续性、职业发展路径、描述清晰度
   - 评分：0-100

3. **岗位相关性 (30%)**
   - 检查：相关经验、相关技能、行业匹配
   - 评分：0-100

**综合质量分**:
```
质量得分 = 完整度 × 0.4 + 逻辑性 × 0.3 + 相关性 × 0.3
```

**应用场景**:
- 影响综合评分的"简历质量"维度 (15%)
- 帮助HR快速识别简历质量
- 提供改进建议

---

### 功能4: 综合评分算法

#### 4.1 评分模型

```
综合评分 = 测评分 × 40%
         + 岗位匹配分 × 30%
         + 完整度加成 × 15%
         + 简历质量分 × 15%
```

**维度详解**:

1. **测评分 (40%)** - 加权平均
   - MBTI: 40%
   - DISC: 30%
   - EPQ: 30%
   - 归一化处理（如只做部分测评）

2. **岗位匹配分 (30%)** - AI算法
   - 能力维度匹配
   - 性格特质匹配
   - 经验背景匹配

3. **完整度加成 (15%)** - 数据全面性
   - 单测评: 65分
   - 双测评: 75分
   - 三测评: 85分
   - 有简历: +5分

4. **简历质量分 (15%)** - 智能评估
   - 无简历: 0分
   - 有简历: 40-95分 (动态评估)

#### 4.2 评分透明化

**评分详情弹窗**:
- 显示4个维度的具体分数
- 展示加权计算过程
- 提供评分说明
- 支持查看交叉验证结果

**设计理念**:
- ✅ 评分可解释
- ✅ HR可理解
- ✅ 决策有依据

## 🔧 技术架构特色

### 1. 多模型分层路由

**设计理念**: 根据场景自动选择最优模型

```
请求进入
    ↓
判断分析级别
    ├─ Expert → DeepSeek-R1 (深度推理)
    ├─ Pro → Qwen2.5-32B (默认)
    └─ Normal → Qwen2.5-7B (兜底)
    ↓
调用对应模型
    ↓
统一返回格式
```

**优势**:
- ✅ 成本可控：大部分场景用32B即可
- ✅ 质量保证：关键场景用DeepSeek-R1
- ✅ 速度优化：快速场景用7B兜底

### 2. 智能缓存机制

**缓存策略**:
- 按分析级别缓存（深度/专家分别缓存）
- 数据版本控制（测评/简历变更自动失效）
- 有效期管理（7天自动过期）

**性能提升**:
- 首次生成: 7-15秒
- 二次查看: <1秒
- 节省成本: 80%+ API调用

### 3. 降级保护机制

**场景1: AI超时/失败**
```
AI调用失败
    ↓
自动切换到规则引擎
    ↓
基于测评数据生成基础分析
    ↓
显示"规则分析"徽章
```

**场景2: 数据不足**
```
无测评数据
    ↓
提示候选人完成测评
    ↓
不生成画像
```

**用户体验**:
- ✅ 系统始终可用
- ✅ 降级对用户透明
- ✅ 提供重试选项
